I will write a paragraph for each methodology of SDLC. There are many of them but I will suggest a few:

  1. Waterfall Model: This is a linear and sequential approach where each phase must be completed before the next phase begins. It is easy to understand and manage but lacks flexibility.
  2. V-Model: Also known as the Verification and Validation model, it is an extension of the Waterfall model. Each development stage is associated with a testing phase, ensuring high quality but can be rigid and less adaptive to changes.
  3. Iterative Model: This model focuses on repetition, allowing developers to create a basic version of the software and then iteratively improve it. It is flexible and allows for changes, but can be resource-intensive.

Among so many SDLC Methodologies the ones that are in the AGILE classification are better. Among AGILE SDLC Methods, I have also a few suggestions:

  1. Scrum: A framework that allows teams to work on complex projects and deliver high-value products by breaking down the work into manageable chunks called sprints.
  2. Kanban: A visual approach to managing tasks and workflows, focusing on continuous delivery without overburdening the development team.
  3. Extreme Programming (XP): Emphasizes customer satisfaction and promotes frequent releases in short development cycles, improving productivity and introducing checkpoints to adapt to new customer requirements.
  4. Lean Development: Focuses on delivering value to the customer by eliminating waste, improving efficiency, and continuously improving processes.

I have used CSS in this webpage and there are 3 ways to include CSS in a webpage as follows:

  1. Inline CSS: This method involves adding CSS directly to the HTML elements using the "style" attribute.
  2. Internal CSS: This method involves adding CSS within a "style" tag inside the "head" section of the HTML document.
  3. External CSS: This method involves linking an external CSS file to the HTML document using the "link" tag.

The reasons CSS used in a webpage are:

  1. Separation of Content and Design: CSS allows you to separate the content of your webpage (HTML) from its design (CSS), making it easier to maintain and update.
  2. Consistency: CSS enables you to apply consistent styling across multiple pages of a website, ensuring a uniform look and feel.
  3. Improved Load Times: By using external CSS files, you can reduce the amount of code in your HTML files, leading to faster load times.

We also use JavaScript in webpage development. The main reasons are as follows:

  1. Interactivity: Adds dynamic features like forms, buttons, and pop-ups.
  2. User Experience: Enhances the site with smooth animations and transitions.
  3. Asynchronous Communication (AJAX): Updates page content without reloading.
  4. Cross-Browser Compatibility: Ensures the site works across different browsers and devices.
